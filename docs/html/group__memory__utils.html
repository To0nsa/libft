Detailed Description

Functions for low-level memory operations.

This group contains functions that replicate or extend common C standard library memory operations, including:

  • Allocation (ft_calloc, ft_realloc)
  • Initialization (ft_bzero, ft_memset)
  • Copying and moving (ft_memcpy, ft_memmove)
  • Comparison and search (ft_memcmp, ft_memchr)

These utilities provide safer or more convenient alternatives to their standard counterparts and are useful throughout custom C projects.

This group includes:

  • ft_calloc
  • ft_realloc
  • ft_bzero
  • ft_memchr
  • ft_memcmp

Sets a block of memory to zero.

This function sets the first n bytes of the memory area pointed to by s to zero (bytes containing '\0'). It is commonly used to initialize buffers or memory blocks to a known cleared state.

Parameters
sPointer to the memory area to zero out.
nNumber of bytes to set to zero.
Returns
Nothing.
Note
This function behaves similarly to the standard bzero function. It is often used in combination with malloc, calloc, or custom memory allocators. ft_memset is preferred.

Allocates and zero-initializes memory for an array.

This function allocates memory for an array of nmemb elements of size bytes each and initializes all bytes in the allocated memory to zero. It also includes overflow protection for nmemb * size.

Parameters
nmembNumber of elements to allocate.
sizeSize in bytes of each element.
Returns
A pointer to the allocated memory, or NULL if the allocation fails or if an overflow is detected.
Note
The returned memory must be freed using free when no longer used.

Scans memory for the first occurrence of a specific byte.

This function scans the initial n bytes of the memory area pointed to by s for the first instance of the byte c (converted to an unsigned char). If the byte is found, a pointer to the matching byte is returned.

Parameters
sPointer to the memory block to be scanned.
cByte to search for, interpreted as unsigned char.
nNumber of bytes to scan.
Returns
A pointer to the matching byte in memory, or NULL if the byte is not found in the first n bytes.

Compares two memory blocks.

This function compares the first n bytes of memory areas s1 and s2. The comparison is done byte-by-byte using unsigned characters.

Parameters
s1Pointer to the first memory block.
s2Pointer to the second memory block.
nNumber of bytes to compare.
Returns
An integer less than, equal to, or greater than zero if the first n bytes of s1 are found, respectively, to be less than, to match, or be greater than those of s2.
Note
The comparison stops at the first differing byte or after n bytes.

Copies n bytes from src to dest.

This function performs a memory copy from the source pointer src to the destination pointer dest. It copies 64-bit chunks when possible for performance, then completes with byte-by-byte copying.

Copies n bytes from src to dest, handling overlap.

This function behaves similarly to ft_memcpy, but is safe to use when the source and destination memory areas overlap. It chooses forward or backward copying direction based on pointer addresses.

Parameters
destDestination memory area.
srcSource memory area.
nNumber of bytes to copy.
Returns
A pointer to the destination memory area dest.
Note
If n is zero, the function returns dest immediately. If memory regions overlap, the function copies from end to start to prevent data corruption.

size_t  old_size,
size_t  new_size 
)

Reallocates a memory block to a new size.

Allocates a new block of memory with size new_size, copies up to old_size bytes from the original ptr into the new block, and frees the old memory.

If ptr is NULL, this function behaves like malloc(new_size). If new_size is 0, frees the memory and returns NULL. If allocation fails, returns NULL and the original memory is not freed.

Parameters
ptrPointer to the previously allocated memory block.
old_sizeThe size of the old memory block in bytes.
new_sizeThe desired new size in bytes.
Returns
A pointer to the new memory block, or NULL on failure.
Note
The content beyond the old size is uninitialized.
See also
ft_memmove
